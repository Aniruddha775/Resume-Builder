# Phase 2 — UI Design Contract: Resume Editor & PDF

> Visual and interaction contract for the resume editor and live PDF preview.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (CLI v4) |
| Preset | style: base-nova, baseColor: neutral, cssVariables: true |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react |
| Font | Inter (var(--font-inter) → --font-sans) |

**Source:** `components.json` in worktree — shadcn already initialized in Phase 1.

Available Phase 1 components (free to use, no new installs needed):
`resizable`, `tabs`, `separator`, `tooltip`, `button`

New components to add this phase: `input`, `label`, `textarea`, `badge` (for skill tags).

No third-party registries declared.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding, skill tag gap |
| sm | 8px | Input field vertical padding, section divider gap, icon-to-label gap |
| md | 16px | Panel horizontal padding (matches Phase 1: `px-4`), form field spacing |
| lg | 24px | Section block spacing within editor (between contact form and Tiptap area) |
| xl | 32px | Spacing between major resume sections in the PDF template |
| 2xl | 48px | Not used in editor UI; reserved for PDF page top/bottom margin |
| 3xl | 64px | Not used |

Exceptions:
- Panel heading row: `px-4 py-3` (matches Phase 1 established pattern — 12px vertical, 16px horizontal)
- Touch targets for delete icons: minimum 32x32px hit area, visually 16x16px icon
- "Add Entry" buttons: full width within panel, 8px vertical padding
- PDF page margins: 48px top/bottom, 40px left/right (within @react-pdf/renderer)

## Typography

### App UI Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (form inputs, editor text) | 14px | 400 | 1.5 |
| Label (input labels, section headings within forms) | 13px | 600 | 1.4 |
| Panel heading (matches Phase 1: "Resume Editor", "Preview") | 13px | 600 | 1.4 |
| Skill tag text | 12px | 400 | 1.4 |

**Source:** Phase 1 established `text-[13px] font-semibold` for panel headings. App UI stays tight — no display sizes needed in editor panels.

### PDF Template Typography (Inter via @react-pdf/renderer)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Candidate name (heading) | 22pt | 700 | 1.2 |
| Section heading (EXPERIENCE, EDUCATION, SKILLS) | 11pt | 700 | 1.2 |
| Body text (company, role, institution, degree, bullet text) | 10pt | 400 | 1.5 |
| Metadata (dates, location, contact info line) | 9pt | 400 | 1.4 |

**Note:** @react-pdf/renderer uses pt units. 10pt = ~13.3px at 96dpi. Inter is available via the `@fontsource/inter` package or loaded as a URL in the pdf Document component.

## Color

All values from established Phase 1 CSS custom properties in `globals.css`. Light mode only. No dark mode in Phase 2.

| Role | Value (CSS token) | Resolved (OKLCH) | Usage |
|------|-------------------|-----------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) = white | App background, editor area background, form fields |
| Secondary (30%) | `--muted` / `--sidebar` | oklch(0.97 0 0) = near-white gray | Panel header rows (`bg-muted`), right panel PDF preview background, skill tag chips background |
| Accent (10%) | `--primary` | oklch(0.205 0 0) = near-black | Reserved — see below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) = red | Delete entry icon hover state only |

**Accent (`--primary`) reserved strictly for:**
1. "Export PDF" button (primary action, phase CTA)
2. "Add Position" / "Add Education" / "Add Skill Group" buttons — ghost or outline variant, NOT filled
3. Active tab indicator underline in mobile layout (`border-primary`)
4. Focused input ring (`--ring` = oklch(0.708 0 0))

**"Saved ✓" indicator:** Use `--color-success` (oklch(0.723 0.191 142.5) = green) for the checkmark icon. The chip background uses `--muted`. Text: `--muted-foreground`. Duration: 2 seconds, then auto-hides.

**PDF template colors:** Black text (#000000) on white (#FFFFFF) only. No accent colors inside the PDF. Section heading rule: 0.5pt solid `#000000` horizontal line under each section heading.

**Borders:** `--border` (oklch(0.922 0 0) = light gray) for all panel dividers, form input borders, and section separators.

## Component Inventory

### Editor Panel Components (new in Phase 2)

**ContactForm** — Plain `<form>` with `<label>` + shadcn `<Input>` pairs. Not Tiptap.
- Fields: Full Name, Email, Phone, Location, LinkedIn (optional), Website (optional)
- Layout: 2-column grid at md breakpoint, single column below
- All inputs: `text-sm`, `h-9`, border from shadcn input defaults

**SectionHeading** — Reusable div inside editor panel with section label and "Add Entry" button side-by-side.
- Label: 13px / 600 / `text-foreground`
- "Add Entry" button: `variant="ghost"` or `variant="outline"`, `size="sm"`, icon-leading (Plus from lucide)

**ExperienceEntry** — Card-like block, no border/shadow, `bg-background`. On hover: shows `<DeleteButton>` (Trash2 icon, 32px hit area, `text-destructive` on hover).
- Company: `<Input>` 14px/400
- Title: `<Input>` 14px/400
- Start / End dates: two side-by-side `<Input type="text">` with placeholder "YYYY-MM"; "Current" `<Checkbox>` disables end date
- Bullets: Rendered inside Tiptap BulletList node for the entry scope

**EducationEntry** — Same delete-on-hover pattern as ExperienceEntry.
- Institution: `<Input>`
- Degree + Field: two `<Input>` fields side-by-side
- Graduation: `<Input type="text">` placeholder "YYYY-MM"
- GPA: `<Input>` optional, placeholder "e.g. 3.8"

**SkillGroup** — One skill category per group.
- Category label: `<Input>` placeholder "Category (e.g. Languages)"
- Skills input: comma-separated `<Input>` that renders parsed values as `<Badge variant="secondary">` chips below. Pressing comma or Enter tokenizes. Clicking a chip's X removes the item.
- Delete group: same Trash2 icon on hover (32px hit area)

**Tiptap Editor Area** — Single `<Tiptap>` instance (Tiptap v3 JSX component).
- Wraps Summary section (freeform `<textarea>`-like prose) and Experience / Education bullet lists
- Contact fields and dates are outside Tiptap (D-05)
- `immediatelyRender: false` to prevent SSR hydration mismatch
- `shouldRerenderOnTransaction: true`
- Focus ring: `outline-2 outline-ring/50` (inherits global `@layer base` rule)
- Min height: 120px for Summary section

**AddEntryButton** — Full-width ghost button at the bottom of each section list.
- Icons: `<Plus />` 16px, leading
- Labels: "Add Position", "Add Education", "Add Skill Group"
- Spacing: `mt-8` above the button, `mb-4` below

**DeleteButton** — Shown on parent entry hover only (CSS `group-hover:visible`, initially `invisible`).
- Icon: `<Trash2 />` 16px, color `text-muted-foreground`, hover color `text-destructive`
- Hit area: `w-8 h-8` (32px) centered around icon
- No confirmation dialog — immediate delete (D-09)
- Tooltip: "Delete" on hover via shadcn `<Tooltip>`

### Right Panel: PDF Preview

**PdfPreviewPanel** — Replaces the "ATS Score" placeholder (D-01, D-02).
- Panel heading changes from "ATS Score" → "Preview" (13px / 600 / `text-foreground`)
- Background: `bg-muted` matching secondary surface
- PDF rendered via `@react-pdf/renderer`'s `pdf().toBlob()` (imperative, not in React tree — PDF-02)
- Blob converted to object URL, displayed in `<react-pdf Document>` viewer
- Debounce: 800ms after last resume state change before re-rendering PDF (PDF-01)
- Loading state during re-render: faint `opacity-50` overlay on the preview + `<Loader2 className="animate-spin" />` centered

**ExportPdfButton** — Lives in panel heading row, right-aligned.
- `variant="default"` (filled primary), `size="sm"`
- Label: "Export PDF"
- Icon: `<Download />` 16px, leading
- On click: triggers blob download, no paywall, no watermark (PDF-05)
- Disabled state: `disabled` while PDF is regenerating

### Header: Autosave Indicator (D-15)

**SavedIndicator** — Inline chip, right side of AppHeader, auto-hides after 2 seconds.
- Structure: `<span>` with `<Check className="text-success" />` + "Saved" text
- Text: 12px / 400 / `text-muted-foreground`
- Icon: Check from lucide, 14px, `text-success` (green)
- Visible only while autosave in-progress flash; renders `null` otherwise

### Mobile: Preview Tab (D-01 mobile)

Repurpose the existing "Score" tab to "Preview" for Phase 2. Tab label: "Preview". The Score tab returns in Phase 3 when scoring is added. Three tabs total: JD | Editor | Preview.

## PDF Template Contract (D-12, D-13, D-14)

**Template name:** Modern Clean (the one ATS-friendly template for Phase 2)

| Property | Value |
|----------|-------|
| Page size | Letter (8.5" × 11") |
| Margins | 48pt top/bottom, 40pt left/right |
| Font family | Inter (loaded via URL or @fontsource) |
| Layout | Single column, no tables |
| Section heading style | 11pt bold ALL CAPS + 0.5pt horizontal rule below |
| Name | 22pt bold, first line |
| Contact line | 9pt regular, one line separated by " • " bullets |
| Body text | 10pt regular, 1.5 line height |
| Date metadata | 9pt regular, right-aligned on same row as company/institution |
| Bullet indent | 8pt left indent from body margin |
| Section gap | 16pt between sections |
| Color | Black (#000000) text, white (#FFFFFF) background |
| Images | None |
| Links | Visible URL text only (no hyperlink decoration — ATS parsers ignore href) |

## Interaction States

### Input Fields

| State | Appearance |
|-------|-----------|
| Default | `border-input`, `bg-background` |
| Focus | `ring-2 ring-ring/50 border-ring` |
| Disabled (End date when "Current" checked) | `opacity-50 cursor-not-allowed bg-muted` |
| Error (invalid date format) | `border-destructive ring-destructive/30` — no inline error message in Phase 2; Phase 4 adds validation UX |

### Entry Delete

| State | Appearance |
|-------|-----------|
| Entry default | Delete icon `invisible` |
| Entry hover (group-hover) | Delete icon becomes `visible`, `text-muted-foreground` |
| Delete icon hover | `text-destructive` |
| Delete icon click | Entry removed immediately; no confirmation |

### PDF Export Button

| State | Appearance |
|-------|-----------|
| Default | `variant="default"` filled, enabled |
| While regenerating | `disabled`, `opacity-50` |
| Download triggered | Brief text change to "Downloading..." for 1s, then resets |

### Autosave Indicator

| State | Appearance |
|-------|-----------|
| Idle | Not rendered |
| Saving (10s interval fires) | Chip appears with Check icon + "Saved" |
| After 2 seconds | Chip disappears (CSS transition: `opacity 0 over 300ms`) |

### Skill Tag Input

| State | Appearance |
|-------|-----------|
| Typing | Raw text visible in `<Input>` |
| After comma or Enter | Token becomes `<Badge variant="secondary">` chip below field |
| Chip hover | X button appears on chip |
| X click | Chip removed, value removed from `items[]` array |
