   ```typescript
   import type { StateCreator } from 'zustand'
   import type { AppState } from '../index'
   import type { Resume } from '@/types/resume'

   export interface ResumeSlice {
     resume: Resume | null
     setResume: (resume: Resume) => void
     updateBullet: (expIndex: number, bulletIndex: number, text: string) => void
     clearResume: () => void
   }

   export const createResumeSlice: StateCreator<
     AppState,
     [['zustand/immer', never], ['zustand/devtools', never]],
     [],
     ResumeSlice
   > = (set) => ({
     resume: null,
     setResume: (resume) => set((state) => { state.resume = resume }),
     updateBullet: (expIndex, bulletIndex, text) =>
       set((state) => {
         if (state.resume?.sections.experience[expIndex]) {
           state.resume.sections.experience[expIndex].bullets[bulletIndex] = text
         }
       }),
     clearResume: () => set((state) => { state.resume = null }),
   })
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage -> application | Untrusted data read from browser storage (could be corrupt, tampered, or from an older schema version) |
| JSON.parse -> application | Malformed JSON strings from localStorage could throw |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | localStorage reads | mitigate | All reads validated with Zod safeParse. Invalid data returns null, never crashes. Implemented in LocalStorageAdapter.getResume() and getJobDescription(). |
| T-02-02 | Denial of Service | localStorage quota | accept | Phase 1 has minimal data. ERRH-03 (Phase 4) adds storage-full detection with JSON export backup. |
| T-02-03 | Information Disclosure | localStorage data | accept | All data is user-owned resume content on their own browser. No sensitive secrets stored in Phase 1. API keys stored in Phase 5 will use encryption. |
| T-02-04 | Tampering | Zustand persist rehydration | mitigate | persist middleware has version field (version: 1) for future migrations. Zod validation on storage reads catches any tampered persisted state. |
| T-02-05 | Spoofing | JSON.parse injection | mitigate | All JSON.parse calls wrapped in try/catch. Malformed JSON returns null, never propagates. |
</threat_model>
