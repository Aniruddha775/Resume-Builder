## Spacing Scale

Declared values (multiples of 4, 8-point base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight element spacing |
| sm | 8px | Compact element spacing, chip gaps, tab padding |
| md | 16px | Default element spacing, panel internal padding |
| lg | 24px | Section padding within panels, card padding |
| xl | 32px | Gap between major layout sections |
| 2xl | 48px | Panel header area height (padding included) |
| 3xl | 64px | Page-level vertical spacing (e.g., above app header if added later) |

Exceptions: Panel resize handles use 8px (sm) hit target width with 1px visible divider line centered within. The `react-resizable-panels` `PanelResizeHandle` element should be 8px wide for pointer precision.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Placeholder instructional text, panel body content |
| Label | 13px | 600 (semibold) | 1.4 | Panel headers ("Job Description", "Resume Editor", "ATS Score"), tab labels |
| Heading | 20px | 600 (semibold) | 1.2 | App title "Rese" in top bar, section headings |
| Display | 28px | 600 (semibold) | 1.2 | Reserved for future score display (Phase 3). Not used in Phase 1. |

**Font family:** Inter, loaded via `next/font/google` with `subsets: ['latin']`, `weight: ['400', '600']`. Applied as a CSS variable `--font-inter` on the `<html>` element, referenced in Tailwind CSS `@theme` as the default sans-serif.

**Rationale:** 14px body is standard for productivity tools (Linear uses 14px body, Notion uses 14-16px). 13px labels create visual hierarchy without feeling cramped. Two weights only (400 + 600) keeps typography disciplined.

## Color

All colors use shadcn/Tailwind CSS theme variables. Values below are the resolved HSL equivalents for light mode.

| Role | CSS Variable | Resolved Value | Usage |
|------|-------------|----------------|-------|
| Dominant (60%) | `--background` | hsl(0 0% 100%) / #FFFFFF | Page background, panel backgrounds |
| Secondary (30%) | `--muted` | hsl(210 40% 96%) / #F4F6F8 | Panel chrome background, tab bar background, resize handle hover |
| Border | `--border` | hsl(214 32% 91%) / #E2E8F0 | Panel dividers, card borders, separator lines |
| Foreground | `--foreground` | hsl(222 47% 11%) / #0F172A | Primary text (headings, body text) |
| Muted foreground | `--muted-foreground` | hsl(215 16% 47%) / #64748B | Placeholder instructional text, secondary labels |
| Accent (10%) | `--primary` | hsl(222 47% 11%) / #0F172A | Reserved for: active tab indicator, focused resize handle |
| Destructive | `--destructive` | hsl(0 84% 60%) / #EF4444 | Not used in Phase 1. Reserved for future delete confirmations. |

**Accent reserved for (Phase 1 only):**
- Active tab underline/indicator on mobile tabbed view
- Keyboard focus ring on interactive elements (2px solid, 2px offset)
- Resize handle on hover/active state

**NOT used for accent in Phase 1:** There are no buttons, links, or CTAs in Phase 1. Accent usage expands in Phase 2+ as interactive elements are added.

**Functional colors (defined now, used in Phase 3+):**
- `--success`: hsl(142 71% 45%) / #22C55E — Matched keywords (green)
- `--warning`: hsl(38 92% 50%) / #F59E0B — Partial matches
- `--danger`: hsl(0 84% 60%) / #EF4444 — Missing keywords (red)

These are defined in the theme layer now for consistency but have no Phase 1 usage. Source: DESIGN.md wireframe specifies green for matched, red for missing keywords.

| Panel order | Left: JD Input, Center: Resume Editor, Right: ATS Score | DESIGN.md wireframe |

## Interaction Contract

### Resize Handles (Desktop)

| Interaction | Behavior |
|-------------|----------|
| Hover | Handle line changes from `--border` to `--primary`, cursor changes to `col-resize` |
| Drag | Panel widths adjust proportionally. Minimum width constraints enforced (200px / 300px / 200px). |
| Double-click | Reset panels to default proportions (27% / 46% / 27%) |
| Keyboard | Not required in Phase 1. Keyboard panel navigation deferred to Phase 4 (A11Y). |

### Tabs (Mobile/Tablet)

| Interaction | Behavior |
|-------------|----------|
| Tap/click tab | Switch active panel instantly. No animation. |
| Active indicator | 2px bottom border in `--primary` color |
| Focus ring | 2px solid `--primary`, 2px offset on keyboard focus |

### Scroll

| Interaction | Behavior |
|-------------|----------|
| Panel overflow | Each panel scrolls independently via `overflow-y: auto` |
| Scrollbar style | Native scrollbar. No custom styling in Phase 1. |

## Accessibility Baseline (Phase 1)

Full WCAG 2.1 AA compliance is Phase 4, but Phase 1 establishes the foundation:

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | `<header>`, `<main>`, `<section>` with `aria-label` per panel |
| Tab order | Left panel, center panel, right panel (natural DOM order) |
| Focus visible | 2px solid `--primary`, 2px offset on all focusable elements |
| Color contrast | All text meets 4.5:1 minimum against its background (verified: #0F172A on #FFFFFF = 15.4:1, #64748B on #FFFFFF = 4.6:1) |
| Reduced motion | `prefers-reduced-motion: reduce` disables any transition on resize handles |
| Panel landmarks | Each panel is a `<section>` with `aria-label="Job Description"`, `aria-label="Resume Editor"`, `aria-label="ATS Score"` |

## Tailwind CSS v4 Theme Configuration

Since Tailwind CSS v4 uses CSS-first configuration (no `tailwind.config.js`), the design tokens above are applied via `@theme` in the global CSS file:

```css
/* src/app/globals.css */
@import "tailwindcss";

@theme {
  --font-sans: "Inter", ui-sans-serif, system-ui, sans-serif;
  --color-success: oklch(0.723 0.191 142.5);
  --color-warning: oklch(0.768 0.165 54.0);
  --color-danger: oklch(0.577 0.245 27.3);
}
```

shadcn CSS variables (`--background`, `--foreground`, `--primary`, `--muted`, `--border`, `--destructive`, etc.) are managed by shadcn's own CSS layer and should NOT be overridden in `@theme`. They live in the `:root` block that shadcn generates.
